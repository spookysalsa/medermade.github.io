What is Pair Programming?

It's collaborating on coding together; Wherein, one person drives and the other navigates. We will explain what the differences between Driver and Navigator. It can also be described as the process of pairing up on a given project with another person so as to devide the duties envolved. This manner of coding creates a space where each person can dedicate their attention and energy on one aspect but with the same purpose in mind. It creates space in a world where a single mind can feel cramped with too much to hold on to.

Why is Pair Programming Important?

Because it allows programmers to devide responsibilities between eachother. It allows the driver to focus on the actual process of inputting the code and allows the navigator to focus on how the code will fit and function in the entirity of the program.

Compare Navigator and Driver?

There is a differnce in Driver and Navigator. The Driver takes on the responsibility of typing the code into a document. The Driver should be well aware of the code (language) needed for the type of document. Given that the Driver is well versed in the appropriate code, the Navigator's role is to communicate what will be in the content of the document. At times it is appropriate switch roles.
